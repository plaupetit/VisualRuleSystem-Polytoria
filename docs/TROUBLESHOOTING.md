# VRS Troubleshooting

This page lists common problems and the first checks to run.

## Bridge Not Linked

Symptoms:

- Project badge says the project is not linked.
- Creator hierarchy and file browser do not refresh.
- Deploy buttons are disabled.

Checks:

- Confirm `active-polytoria-project.json` exists locally and points to the
  Polytoria project folder.
- Confirm the Creator bridge addon is installed.
- Reopen Polytoria Creator after installing the addon.

## Creator Off or Addon Stale

Symptoms:

- Badge says `Creator off`, `Addon stale`, or `Live paused`.
- Creator hierarchy data is old.
- Bridge commands do not apply.

Checks:

- Open the target project in Polytoria Creator.
- Confirm the VRS window has focus when testing live flows that require focus.
- Watch the `Addon beat` badge. A recent beat means VRS can see addon status.

## Deploy File Is Disabled

Symptoms:

- `Deploy File` cannot be clicked.
- The file chip still shows an old path.

Checks:

- Confirm Creator and the bridge are ready.
- Fix validation errors in the `Output: F11` overlay.
- Confirm the graph script kind is correct.

## Deploy Instance Says File Missing or Empty

Symptoms:

- Hierarchy deploy says to deploy the file first.
- The expected `.luau` exists but is 0 bytes.

Fix:

1. Click `Deploy File`.
2. Confirm the file exists under `scripts/VRS/<kind>/`.
3. Deploy the saved instance again from the Creator Hierarchy.

The instance deploy should not generate source code. It only links the saved
file into Creator.

## Trigger Is Silent

Symptoms:

- The script exists in Creator.
- Nothing happens at runtime.
- No useful console output appears.

Checks:

- Enable `Debug logs` on the trigger.
- Confirm the script is deployed as the correct kind: server, client, or module.
- Confirm the object supports the event being listened to.
- For touch triggers, confirm collision/touch behavior is enabled in Creator.
- For player actions, confirm the action parameter uses `Triggering Player`
  when the player comes from the trigger context.

## Script Is Linked but Not Updated

Symptoms:

- Creator has the script instance.
- Runtime behavior looks like an old graph.

Fix:

1. Click `Deploy File` to update the saved `.luau`.
2. Use the File Browser to confirm the saved file changed.
3. If the instance link is missing or points to the wrong file, right-click the
   target in Creator Hierarchy and choose `Deploy Saved Script Instance Here`.

## Cannot Load a Script Back Into VRS

Symptoms:

- `Load VRS Graph From File` does not load the graph.
- VRS says no metadata was found.

Reason:

VRS only loads scripts generated by VRS that contain embedded graph metadata.
Plain Luau scripts are not imported as visual graphs in this version.

## Output Overlay Blocks Mouse Input

Symptoms:

- The output overlay is open and mouse clicks do not reach the canvas.

Fix:

- Press `F11` or `Esc` to close the overlay.
- Or switch the overlay mouse mode to click-through from the `Output: F11`
  controls.
